1. При обговоренні змінних, які знаходяться в скоупі самого класу (змінні-поля, ми інколи називаємо їх “глобальними”,
хоча це некоректна назва), ми підкреслили, що вони мають бути “останнім засобом” для обміну даними в програмі. Майже
завжди краще використати параметри, які передаються у функцію і результат який вона повертає. Для кожної змінної в
програмі поясніть, чому ви вибрали створити її не-локально
(константи не вважаються змінними, тому не треба описувати константи тут).

Клас Ball
Змінні:
- vx та vy - безпосередньо характеризують об'єкт м'яча (його швидкість по осях Х та У). Вони використовуються у багатьох
методах, що працюють з м'ячем. Зробити їх полями дозволяє зберігати і змінювати стан м'яча протягом гри, подібно до
його розмірів чи координат.

Клас Breakout
Змінні:
- ui - об'єкт для взаємодії з елементами інтерфейсу. Використовується в методі mouseMoved, який не приймає
інших параметрів, окрім події.

Клас GameController
Змінні:
- animationPause - пауза між оновленнями анімації. Використовується в методі, який викликається після ланцюга інших
методів. Вирішила, що чистіше буде зробити її доступною для усіх методів, ніж передавати як параметр через кілька
проміжних викликів, де вона не використовується.
- ui - об'єкт для взаємодії з елементами інтерфейсу. Використовується в усіх методах класу, тому доцільно зробити його
полем, щоб кожен раз не передавати як аргумент з методу в метод.
- gameState - об'єкт, що зберігає поточний стан гри. Використовується в багатьох методах класу, тому зручніше зробити
його полем, ніж передавати через ланцюжок викликів.

Клас GameState
Змінні:
- bricksLeft - кількість цеглинок, що залишились на екрані.
- isWin - маркер, що позначає чи виграв гравець.
Об'єднала їх в одному класі, щоб можна було передавати стан гри як єдиний об'єкт.

Клас GameUIManager
Змінні:
- window - об'єкт WindowProgram з головного методу.
- paddle - об'єкт GReact з розширеною функціональністю.
- ball - об'єкт GOval з розширеною функціональністю.
- infoLabel - об'єкт GLabel для відображення тексту.
- resultLabel - об'єкт GLabel для відображення результатів гри.
Об'єднала всі ці змінні в одному класі, щоб мати можливість керувати елементами інтерфейсу через один об'єкт.

Клас Paddle
Змінні:
- fixedY - координата верхньої сторони прямокутника. Зберігається між викликами методу slide, щоб гарантувати стабільне
положення платформи по вертикалі.

Клас PhysicsUtils
- немає змінних, усі методи статичні.

Клас Utils
- немає змінних, усі методи статичні.


2. Гра закінчується або коли користувач втрачає три кулі, або коли уся цегла розбита. Як ви перевіряєте закінчення гри
в цьому другому випадку?
- Я використовую об'єкт GameState, який зберігає стан гри під час виконання. При розбитті кожної цеглинки викликається
метод, що зменшує лічильник bricksLeft і перевіряє, чи залишилась цегла: якщо ні - isWin = true.
Коли програма повертається по ланцюгу викликів, значення isWin перевіряється, щоб у разі перемоги припинити подальше
виконання гри.
